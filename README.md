# fließend

**流れるように。思考とファイルの境界を溶かす。**

## コンセプト

フォルダ構造をアウトライナーのように編集できるツール。
エディターとファイラーの限界を超える。

### 核心的なアイディア

- **左側のペインをテキストのように編集する**
- インデント変えたらフォルダが移動する
- コピペでフォルダ構造を複製する
- Claudeと対話しながら構造を整理する

### なぜ作るのか

満足したことを「こねくり回せる状態」で残したい。
思考を固定せず、流動的に扱いたい。

与太話を膨らませて、いじくり回して、また新しい形にする。
そのための遊び場。

## 技術的な要素

### フォルダとファイルの等価性

- フォルダもファイルも同じように扱う
- 重複名を許容（内部でUUID管理）
- 表示名と実際のフォルダ名を分離

### メタデータ

- `.outline-meta/` で長文やタグを保存
- フォルダに「ノート欄」を持たせる
- 外から見ても普通のフォルダ構造

### AIとの関係

AIにとってフォルダ構造は「コンテキスト境界」。
セマンティック検索よりも、階層で絞ってから検索する方が効率的。

## 類似プロジェクト

- **OutWiker**: ノートツリーをディレクトリとして保存
- **Leo Editor**: 外部ファイルとのライブ同期
- でも、純粋な「ファイルシステム管理のためのアウトライナー」は存在しない

## 使い方

### 起動

```bash
# 依存関係のインストール
npm install

# 開発モードで起動
npm run dev

# 通常モードで起動
npm start
```

### 基本操作

1. **フォルダを開く**: 「フォルダを開く」ボタンをクリック
2. **ナビゲーション**: ↑↓キーでファイル/フォルダ間を移動
3. **フォルダの開閉**: Enter/Spaceキーまたはクリック
4. **階層の変更**: **Tab/Shift+Tabでファイル/フォルダの階層を変更**
5. **ファイルを開く**: ダブルクリックで外部エディタで開く

### キーボードショートカット

**ナビゲーション:**
- `↑/↓`: 選択を上下に移動
- `←/→`: フォルダを閉じる/開く
- `Enter` / `Space`: フォルダを開閉

**階層操作（コア機能）:**
- `Tab`: 選択したノードの階層を下げる（直前の兄弟フォルダの子にする）
- `Shift+Tab`: 選択したノードの階層を上げる（親の兄弟にする）

**制限事項:**
- ルートフォルダは移動できません
- ファイルの下には移動できません
- 同名のファイル/フォルダがある場所には移動できません
