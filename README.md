# fließend

**流れるように。思考とファイルの境界を溶かす。**

## コンセプト

フォルダ構造をアウトライナーのように編集できるツール。
エディターとファイラーの限界を超える。

### 核心的なアイディア

- **左側のペインをテキストのように編集する**
- インデント変えたらフォルダが移動する
- コピペでフォルダ構造を複製する
- Claudeと対話しながら構造を整理する

### なぜ作るのか

満足したことを「こねくり回せる状態」で残したい。
思考を固定せず、流動的に扱いたい。

与太話を膨らませて、いじくり回して、また新しい形にする。
そのための遊び場。

## 技術的な要素

### フォルダとファイルの等価性

- フォルダもファイルも同じように扱う
- 重複名を許容（内部でUUID管理）
- 表示名と実際のフォルダ名を分離

### メタデータ

- `.outline-meta/` で長文やタグを保存
- フォルダに「ノート欄」を持たせる
- 外から見ても普通のフォルダ構造

### AIとの関係

AIにとってフォルダ構造は「コンテキスト境界」。
セマンティック検索よりも、階層で絞ってから検索する方が効率的。

## 類似プロジェクト

- **OutWiker**: ノートツリーをディレクトリとして保存
- **Leo Editor**: 外部ファイルとのライブ同期
- でも、純粋な「ファイルシステム管理のためのアウトライナー」は存在しない

## クイックスタート

### 1. インストール

```bash
# 依存関係のインストール
npm install
```

### 2. 起動

```bash
# 開発モード（推奨）
npm run dev
```

### 3. 使ってみる

1. 「フォルダを開く」ボタンをクリック
2. テスト用のフォルダを選択
3. `↑↓` キーでノードを選択
4. `Tab` / `Shift+Tab` でフォルダ階層を変更
5. 実際のフォルダ構造が変わることを確認！

**詳しい使い方は [使い方.md](使い方.md) を参照してください。**

### テスト用フォルダを作成

```bash
# テストフォルダの作成
mkdir -p ~/fliesend-test/{プロジェクトA,プロジェクトB,ドキュメント}
mkdir -p ~/fliesend-test/プロジェクトA/{資料,コード}
mkdir -p ~/fliesend-test/プロジェクトB/{画像,メモ}
touch ~/fliesend-test/README.txt
touch ~/fliesend-test/プロジェクトA/資料/仕様書.md

echo "テストフォルダを作成: ~/fliesend-test"
```

## キーボードショートカット

| キー | 動作 |
|------|------|
| `↑/↓` | ノード選択移動 |
| `←/→` | フォルダ展開/折りたたみ |
| `Space` | フォルダ開閉 |
| **`Tab`** | **ノードを一段階下げる** ⭐ |
| **`Shift+Tab`** | **ノードを一段階上げる** ⭐ |
